Add new Edit

The implementation idea

This should be simple right?

  • Add a text area.

  • When the user clicks on save, an html file is created

  • Then copy the files served by any static page site (gh-pages for instance).

What could go wrong?

With this in mind, we created a backlog in using GitHub project management feature. That way we can divide the work and team members can take an issue and work it out.

We intended to work using a scrum-ish / agile with a week long iterations, initially to be divided as follows:

  • w1 Setup the project and create a scratch version

  • w2 Work on integrating an existing editor and/or give up and don't use one at all

  • w3 Reactify / work on the mobile version, other nice to have thing

  • w4 Security to prevent injection, limits, abuse,  user profiles

  • w5 Wrap up and deploy

So, the issues for the first week were created, and we started to work, doing daily “standups” in slack.

The task for the first week were:

  • Setup environment (npm, js, node)

  • Create a sample html file with the CSS needed from the mockup

  • Search for a markdown processor

  • Create an example file with a <textarea> for input

  • Create a controller that takes that input and save the file.


I would say the first week was very smooth, we were very motivated to start working and everything went as planned.

Then we reached the second week, and thing got a bit complicated.
